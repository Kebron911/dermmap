### WORKFLOW WALKTHROUGH

**Section label**
How it works

**Headline**
From patient selected to lesion documented.

**Step 1 — Select Patient**
Search by name, DOB, or MRN. Or let your EHR schedule pull today's appointments automatically. Recent patients are one tap away.

**Step 2 — Tap the Body Map**
A full anatomical diagram — anterior, posterior, lateral, head detail. Tap where the lesion is. Pinch to zoom for precision. A marker appears. Prior visit markers are visible in muted colors for context.

**Step 3 — Snap a Photo**
The camera opens instantly with a circular framing guide for consistent shots. The photo attaches to the marker automatically. No naming. No filing. No manual linking.

**Step 4 — Document in Seconds**
A bottom sheet slides up with structured fields: size, shape, color, border, symmetry, action. Quick-tap options, not free text. Sensible defaults so the MA can move on. Save and next.

**Step 5 — Visit Complete**
One tap. The visit summary shows what was documented, flags any lesions for biopsy review, and queues the record for the physician to sign off.

## PAGE: Features (detailed)

---

### PAGE HEADLINE
Everything built for one workflow: dermatology documentation in the exam room.

---

**Feature: Anatomical Body Map**
A precise, full-body SVG diagram — anterior, posterior, left and right lateral, head/face detail, hands, and feet. Tap to place a marker at the exact lesion location. Pinch to zoom for precision on small areas. Prior visit markers are visible in a distinct color so MAs can see existing documented lesions at a glance. New markers pulse to confirm placement.

**Feature: Clinical Photo Capture**
The camera opens the instant a marker is placed. A circular framing guide ensures consistent framing across visits — critical for meaningful longitudinal comparison. Auto-capture fires when the image is stable and in-focus (optional). Multiple photos per marker are supported for macro and dermoscopic shots. Photos attach automatically — no manual linking step, ever.

**Feature: Dermoscopy Lens Support**
Compatible with DermLite, 3Gen, and Canfield lens adapters. The app detects the attached lens and adjusts exposure and white balance automatically. Dermoscopic and clinical photos are stored separately and labeled accordingly.

**Feature: Structured Documentation**
A bottom sheet with quick-tap fields: size (1–6mm+ with custom entry), shape, color, border, symmetry, and action (monitor, biopsy scheduled, biopsy performed, excision, referral). Sensible defaults let MAs skip unneeded fields without slowing down. Free-text clinical notes are available to providers only — grayed out at the MA role level by design.

**Feature: Offline Mode**
Full offline operation with local AES-256 encryption. Mutations are queued and replayed automatically on reconnect with conflict resolution. Designed as a first-class feature, not a fallback.

**Feature: Longitudinal Lesion Tracking**
Every lesion builds a complete history: photo carousel, side-by-side comparison, structured data changes over time, biopsy results, and pathology notes. Status indicators show at a glance what's being monitored, what's been biopsied, and what's been removed.

**Feature: PDF Export**
One-tap generation of a PDF/A-compliant clinical document: patient info, visit date, body map with numbered markers, per-lesion photo grid, structured data table, and provider signature line. Clinical-quality photos embedded (not thumbnails). Target file size under 5MB for typical visits. Compatible with every EHR as a document attachment — no API required.

**Feature: Role-Based Access**
Three distinct roles — Medical Assistant, Physician, Practice Manager — each with precisely scoped permissions. No user sees data outside their role. No shared logins. Session timeout with biometric/PIN resume. Break-glass emergency access with mandatory justification logging.

**Feature: Audit Log**
Immutable, searchable log of every system action — created, viewed, updated, exported, logged in, logged out. Retained 6 years. Exportable to CSV. Searchable and filterable by user, date, action type, and affected record.

**Feature: Practice Analytics Dashboard**
Visits per day/week/month, average documentation time, photos per visit, provider adoption metrics, and an ROI calculator that converts visit volume and MA hourly cost into projected time savings.

## PAGE: Security

---

### PAGE HEADLINE
HIPAA compliance you can actually explain to your compliance officer.

**Subheadline**
DermMap's security architecture was designed from the first line of code as a clinical application — not adapted from a consumer product after the fact.

**Body intro**
Every access control, encryption layer, and audit trail in DermMap exists because of a specific HIPAA requirement or clinical data security principle. Here's how it maps.

---

**Security spec table (full)**

| Requirement | DermMap Implementation |
|---|---|
| Encryption at rest | AES-256 for all stored data — patient records, lesion data, photos, audit logs |
| Encryption in transit | TLS 1.3 (minimum TLS 1.2) for all API calls and data transfers |
| Unique user identification | Individual accounts required — shared logins are blocked |
| Automatic logoff | 5-minute timeout on mobile, 15-minute timeout on web; PIN/biometric resume |
| Audit controls | Immutable audit log for every action — user, timestamp, action type, affected record |
| Audit retention | 6 years minimum, per HIPAA requirements |
| Access control (role-based) | MA, Provider, Admin — minimum necessary access principle enforced |
| Emergency access | Break-glass access with mandatory justification logging |
| PHI on device | Photos stored only in encrypted app sandbox — never in camera roll |
| EXIF data | Stripped from all photos before storage (EXIF embeds GPS coordinates — PHI risk) |
| Screen protection | Screenshot and screen recording disabled within the app (iOS/Android) |
| Remote wipe | Enrolled device remote wipe capability for lost/stolen devices |
| BAA | Business Associate Agreement available with HIPAA-eligible cloud infrastructure |
| Infrastructure | Dedicated VPC, no public-facing databases, encrypted backups, point-in-time recovery |
